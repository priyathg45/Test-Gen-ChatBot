PROJECT OVERVIEW
================================================================================

A sophisticated AI-powered chatbot system for querying aluminum product 
information using semantic search and machine learning embeddings. The system
uses natural language processing to understand user queries and retrieve 
relevant products from a CSV database.

✓ Conversation History - Maintains full conversation context
✓ Multiple Query Types - Semantic, category, price range, application filters
✓ Statistics & Metrics - Track chat statistics and system performance
✓ Production Ready     - Error handling, logging, configuration management

CORE COMPONENTS
================================================================================

1. DATA PIPELINE
   ├─ DataLoader: Reads and manages CSV data
   ├─ DataPreprocessor: Cleans and prepares data
   └─ Features: Text cleaning, duplicate removal, normalization

2. MACHINE LEARNING LAYER
   ├─ EmbeddingsManager: Creates semantic embeddings
   │  └─ Model: all-MiniLM-L6-v2 (384-dimensional)
   ├─ Retriever: Performs similarity search
   │  └─ Algorithm: Cosine similarity
   └─ Features: Save/load embeddings, multiple search methods

3. CHATBOT ENGINE
   ├─ AluminiumChatBot: Main conversation logic
   ├─ Features: Response generation, history management, statistics
   └─ Configuration: Adjustable similarity threshold, top-K results

4. API LAYER
   ├─ Flask Application: REST API server
   ├─ Endpoints: Chat, history, products, statistics
   └─ Features: CORS enabled, error handling, auto-initialization

USAGE MODES

MODE 1: INTERACTIVE CLI

Features:

- See statistics (type 'stats')

MODE 2: REST API SERVER
-----------------------
Command: python src/api/app.py
Access: http://localhost:5000

Main Endpoints:
- POST /chat              Send message to chatbot
- GET /history            View conversation history
- POST /clear-history     Clear conversation
- GET /stats              Get statistics
- GET /products           List all products
- GET /products/<id>      Get specific product
- GET /health             Health check

Example:
  curl -X POST http://localhost:5000/chat \
    -H "Content-Type: application/json" \
    -d '{"message": "What aluminum alloys for aerospace?"}'

================================================================================
TECHNOLOGY HIGHLIGHTS
================================================================================

Semantic Embeddings:
- Model: Sentence-BERT (all-MiniLM-L6-v2)
- Dimension: 384
- Performance: ~10ms per text encoding
- Use: Semantic similarity search

Similarity Calculation:
- Algorithm: Cosine Similarity
- Formula: cos(θ) = (A·B) / (|A||B|)
- Library: scikit-learn
- Complexity: O(n) per query

Data Processing:
- Framework: pandas
- Features: 
  ├─ Text cleaning (regex)
  ├─ Duplicate removal
  ├─ Missing value handling
  ├─ Price normalization
  └─ Combined text features

Web Framework:
- Framework: Flask 3.0
- Style: RESTful API
- Features:
  ├─ JSON request/response
  ├─ CORS support
  ├─ Error handling
  └─ Auto-initialization

================================================================================
TROUBLESHOOTING GUIDE
================================================================================

Issue: Model Download Fails
Solution: Pre-download model manually
  python -c "from sentence_transformers import SentenceTransformer; 
             SentenceTransformer('all-MiniLM-L6-v2')"

Issue: Port 5000 Already in Use
Solution: Use different port
  export FLASK_RUN_PORT=5001
  python -m flask run

Issue: Out of Memory
Solution: Use smaller embedding model
  Edit .env: MODEL_NAME=distiluse-base-multilingual-cased-v1

Issue: Slow Response Time
Solution: Check system resources
  - Monitor CPU/memory usage
  - Consider using vector indexing (FAISS)
  - Deploy with multiple workers (Gunicorn)

Issue: Poor Similarity Results
Solution: Adjust threshold and top-K
  - Decrease SIMILARITY_THRESHOLD (e.g., 0.2)
  - Increase TOP_K_RESULTS (e.g., 5)
  - Fine-tune embeddings on domain data
